Movie API Documentation

Welcome to the Movie API! This API allows you to manage movies and user information.

Authentication

Authentication is required for certain endpoints using JSON Web Tokens (JWT) and HTTP authorization.

Login Method: POST

To log in use /login?UserName={username}&Password={password} to get a JWT token.

Endpoints

Users

Update User Information

  • URL: /users/:UserName
  • Method: PUT
  • Description: Update a user's information.
  • Authentication: JWT token required.
  • Request Parameters:
    • UserName: Username of the user to be updated.
  • Request Body:
    • UserName: New username.
    • Password: New password.
    • Email: New email address.
    • Birthday: New birthday.
  • Response:
    • 200 OK: Returns details of the updated user.
    • 500 Internal Server Error: If an error occurs during the update process.

Add User

  • URL: /users
  • Method: POST
  • Description: Add a new user.
  • Request Body:
    • UserName: Username of the new user.
    • Password: Password of the new user.
    • Email: Email address of the new user.
    • Birthday: Birthday of the new user.
  • Response:
    • 201 Created: Returns details of the newly created user.
    • 500 Internal Server Error: If an error occurs during user creation.

Add Movie to User's Favorites

  • URL: /users/:UserName/movies/:MovieID
  • Method: POST
  • Description: Add a movie to a user's list of favorite movies.
  • Authentication: JWT token required.
  • Request Parameters:
    • UserName: Username of the user.
    • MovieID: ID of the movie to be added to favorites.
  • Response:
    • 200 OK: Returns details of the updated user.
    • 500 Internal Server Error: If an error occurs during the update process.

Delete Movie from User's Favorites

  • URL: /users/:UserName/movies/:MovieID
  • Method: DELETE
  • Description: Delete a movie from a user's list of favorite movies.
  • Authentication: JWT token required.
  • Request Parameters:
    • UserName: Username of the user.
    • MovieID: ID of the movie to be deleted from favorites.
  • Response:
    • 200 OK: Returns details of the updated user.
    • 500 Internal Server Error: If an error occurs during the update process.

Delete User

  • URL: /users/:UserName
  • Method: DELETE
  • Description: Delete a user.
  • Authentication: JWT token required.
  • Request Parameters:
    • UserName: Username of the user to be deleted.
  • Response:
    • 200 OK: Indicates successful deletion of the user.
    • 400 Bad Request: If the specified user does not exist.
    • 500 Internal Server Error: If an error occurs during the deletion process.

Movies

Get All Movies

  • URL: /movies
  • Method: GET
  • Description: Get a list of all movies.
  • Authentication: JWT token required.
  • Response:
    • 200 OK: Returns an array of movie objects.
    • 500 Internal Server Error: If an error occurs during the retrieval process.

Get Movie by Title

  • URL: /movies/:Title
  • Method: GET
  • Description: Get details of a specific movie by its title.
  • Authentication: JWT token required.
  • Request Parameters:
    • Title: Title of the movie.
  • Response:
    • 200 OK: Returns details of the movie.
    • 500 Internal Server Error: If an error occurs during the retrieval process.

Get Movies by Genre

  • URL: /movies/genre/:genreName
  • Method: GET
  • Description: Get movies belonging to a specific genre.
  • Authentication: JWT token required.
  • Request Parameters:
    • genreName: Name of the genre.
  • Response:
    • 200 OK: Returns an array of movies of the specified genre.
    • 500 Internal Server Error: If an error occurs during the retrieval process.

Get Movies by Director

  • URL: /movies/directors/:directorName
  • Method: GET
  • Description: Get movies directed by a specific director.
  • Authentication: JWT token required.
  • Request Parameters:
    • directorName: Name of the director.
  • Response:
    • 200 OK: Returns an array of movies directed by the specified director.
    • 500 Internal Server Error: If an error occurs during the retrieval process.

Error Handling

  • 500 Internal Server Error: Indicates an unexpected error occurred on the server.
